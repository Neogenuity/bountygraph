What is BountyGraph?

BountyGraph is a Solana-native escrow protocol for multi-phase bounties. When a DAO or agent wants to hire workers for complex, multi-step projects, BountyGraph ensures:

‚úì Dependency Validation

Phase 2 can't start until Phase 1 is complete. Modeled as a topological DAG.

‚úì Proof-of-Work Receipts

Workers submit cryptographic evidence of completion; stored on-chain permanently.

‚úì Trustless Escrow

Funds locked until verification. No centralized intermediary.

‚úì 3-Tier Dispute Resolution

Deterministic validation, oracle attestation, or governance arbitration.

Core Features (Implemented)

Program-Owned Escrow

Funds held in safe, custody-free PDAs. Transfers via lamport mutation.

Topological DAG Validation

Receipt submission checks all dependencies are marked complete before allowing next phase.

Dispute Resolution

Arbiter (graph authority) settles disputes with creator/worker percentage splits.

REST API + TypeScript SDK

Agents can interact programmatically. No UI required.

Deployment on Devnet

  1. git clone https://github.com/neogenuity/bountygraph.git
  2. cd bountygraph && npm install
  3. solana config set --url https://api.devnet.solana.com
  4. anchor build
  5. anchor deploy --provider.cluster devnet
  6. anchor test --provider.cluster devnet

After deployment, the program ID will be synced into Anchor.toml. You can query account states via solana account <PUBKEY>.
